Frequently Asked Questions

Is Papyrus Papers cloud-first?

No. Papyrus Papers is local-first. Your library database, imported PDFs, notes, search indexes, and backups live on your Mac unless you explicitly export, share, or sync them.

What is the difference between Copy Into Library and Keep PDFs In Place?

Copy Into Library copies the PDF into the managed library storage. Keep PDFs In Place leaves the file where it already lives and stores a security-scoped bookmark. Use Copy if you want Papyrus Papers to own the file lifecycle; use Keep if you already manage PDFs in Finder or a synced folder.

Does Papyrus Papers upload my PDFs automatically?

No. Papyrus Papers only sends files off your Mac when you explicitly enable WebDAV sync, share via the macOS share sheet, or export. Metadata lookup sends identifiers and queries, not your library.

Which metadata services does Papyrus Papers use?

Crossref and OpenAlex for DOI lookup, title repair, and metadata completion. arXiv identifiers are extracted from PDF text. Papyrus Papers also reads embedded XMP metadata directly from PDFs (no network). The resolver registry is pluggable, so additional sources can be added without code changes.

Where is the library stored?

Under ~/Library/Group Containers/group.com.zhendafu.Papyrus/Library/Application Support/. Managed PDFs and the full-text index live in the same App Group container.

Does Papyrus Papers keep backups?

Yes. Rolling local database backups, up to seven snapshots. You can also trigger a manual backup from Settings.

How does sync work?

The built-in sync path is WebDAV. You choose the server, username, password, and remote folder. Papyrus Papers can also coexist with folders synced by other tools.

Does Papyrus Papers need an account?

No Papyrus Papers account is required.

Does Papyrus Papers include AI features?

On supported Macs and locales, Papyrus Papers can call Apple's system language-model APIs (Foundation Models) for background metadata synthesis when an enrichment source returns ambiguous or partial data. Nothing in Papyrus Papers requires AI to function — every user-facing feature works without it, and enrichment falls back to non-AI heuristics on unsupported hardware.

Does Papyrus Papers index my library in Spotlight?

Yes. Core Spotlight integration lets papers appear in macOS system search. This is local, not a cloud service.

Filing a Bug Report

Include enough detail to reproduce the problem:

  • macOS version
  • Papyrus Papers version, commit hash, or build date
  • Whether the PDF was copied into the library or kept in place
  • Step-by-step reproduction instructions
  • Whether sync, export, or metadata enrichment were involved
  • Screenshots or screen recordings when helpful

Logs and Diagnostics

Papyrus Papers writes to the macOS unified logging system. A relevant Console excerpt can help when reporting runtime issues. Also useful:

  • Whether automatic backups were enabled
  • Whether the issue persists after restarting
  • Whether it affects one paper, one project, or the whole library

Sensitive Data

Do not post secrets or private material in a public issue. Avoid including WebDAV passwords, OAuth tokens, private server URLs, or confidential PDFs.

Security Issues

If you believe you found a security issue, avoid posting exploit details publicly. Email zdfu189@gmail.com with a minimal description and we will follow up with a private reporting channel.

Self-Build Support

If building from source, include:

  • Xcode version
  • Whether you modified the project locally
  • The build or test command you ran
  • The first relevant compiler or runtime error
